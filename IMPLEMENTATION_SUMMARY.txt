🚀 COMO COMEÇAR:
════════════════════════════════════════════════════════════════════

1️⃣ CRIAR UMA KEY PAIR EC2:
   - Acesse: AWS Console → EC2 → Key Pairs → Create Key Pair
   - Salve o arquivo .pem em local seguro
   - Note o nome da chave

2️⃣ EDITAR CONFIGURAÇÕES:
   cd infra/environments/dev/
   Editar terraform.tfvars:
   - bucket_name: seu S3 bucket existente
   - ec2_key_name: "nome-da-sua-chave-ec2"  ⚠️ IMPORTANTE

3️⃣ INICIALIZAR TERRAFORM:
   cd infra/
   terraform init

4️⃣ PLANEJAR O DEPLOYMENT:
   terraform plan -var-file="environments/dev/terraform.tfvars"

5️⃣ APLICAR A CONFIGURAÇÃO:
   terraform apply -var-file="environments/dev/terraform.tfvars"

6️⃣ OBTER IP DO SERVIDOR:
   terraform output windrose_server_public_ip

❓ TROUBLESHOOTING:
════════════════════════════════════════════════════════════════════

Se algo não funcionar:

1. Verificar logs de inicialização:
   terraform output windrose_server_instance_id
   aws ec2 get-console-output --instance-id <ID>

2. Verificar Security Group:
   aws ec2 describe-security-groups --group-ids <SG-ID>

3. Reconnectar ao servidor:
   terraform taint aws_instance.windrose_server
   terraform apply

4. Destruir e recriar:
   terraform destroy
   terraform apply
