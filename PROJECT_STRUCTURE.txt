🎯 PRÓXIMOS PASSOS:

PASSO 1: PREPARAÇÃO
   [ ] Instale Terraform (terraform.io)
   [ ] Configure AWS CLI com suas credenciais
   [ ] Crie uma EC2 Key Pair na AWS

PASSO 2: CONFIGURAÇÃO
   [ ] Edite: infra/environments/dev/terraform.tfvars
       └─ Mude ec2_key_name para o nome da sua chave

PASSO 3: DEPLOY
   [ ] cd infra
   [ ] terraform init
   [ ] terraform plan -var-file="environments/dev/terraform.tfvars"
   [ ] terraform apply -var-file="environments/dev/terraform.tfvars"

PASSO 4: VALIDAÇÃO
   [ ] Aguarde 3-5 minutos
   [ ] Execute: terraform output
   [ ] Salve o IP público
   [ ] Teste acesso: http://<IP_PÚBLICO>:8000

💡 INFORMAÇÕES IMPORTANTES:

🔑 CREDENCIAIS:
   - Arquivo de chave EC2: Salve em local seguro (~/.ssh/key.pem)
   - Permissões: chmod 400 ~/.ssh/key.pem
   - Não compartilhe a chave com ninguém

📍 ACESSO:
   - SSH: ssh -i ~/.ssh/key.pem ubuntu@<IP>
   - HTTP: http://<IP>:8000
   - Porta SSH: 22
   - Porta Windrose: 8000

⏱️ TEMPOS:
   - Terraform init: 30-60s
   - EC2 provision: 2-3 min
   - Docker download: 1-2 min
   - Container start: 10-30s
   - Total: ~5-10 minutos

💰 CUSTOS:
   - EC2 t3.micro: ~$0.0104/hora ($7.50/mês)
   - Elastic IP: Gratuito em uso
   - Data transfer: Primeiros 1GB/mês gratuito
   - Free tier: 750h/mês por 12 meses

❓ QUESTÕES FREQUENTES:

P: Como mudo a porta do jogo?
R: Edite windrose_port em terraform.tfvars

P: Como aumento o tamanho da instância?
R: Edite ec2_instance_type (de "t3.micro" para "t3.small", etc)

P: Como adiciono monitoramento?
R: Renomeie cloudwatch_monitoring.tf.disabled para .tf

P: Como faço backup?
R: Use AWS Backup ou crie snapshots automáticos

P: Como faço deploy em produção?
R: Use terraform -var-file="environments/main/terraform.tfvars"

P: Posso usar um domínio customizado?
R: Sim, adicione Route 53 (veja EXTENSIONS.md)
